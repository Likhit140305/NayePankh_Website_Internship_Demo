User Flows

Key journeys mapped for three primary user types.

Flow 1 — Volunteer Registration

Land on homepage
→
Click "Volunteer"
→
Registration form
→
Valid inputs?
→
Success + confirmation email
Error path: Invalid inputs → inline field-level error messages → user corrects → resubmit. On success, confirm what happens next ("We'll email you within 2–3 days") to reduce anxiety and drop-off.

Flow 2 — Donor Journey

Arrives via social media
→
Impact stats on hero
→
Scrolls to Donate section
→
Picks amount
→
Payment gateway
→
80G receipt by email
Trust signals matter here: show "₹500 = 1 month of books" breakdowns to make the donation tangible. 80G tax benefit is a strong conversion driver for Indian donors — mention it before the CTA.

Flow 3 — Student / Parent seeking help

Google search: "free tuition NGO"
→
Programs page
→
Program matches need?
→
Contact page
→
Enquiry submitted
This flow is currently broken on the existing NayePankh site — there is no dedicated "get help" path for beneficiaries. A separate "I need help" CTA alongside the "Volunteer" CTA on the homepage would serve this audience.

Flow 4 — Intern application

Recommendation: Add a dedicated Internships page. NayePankh runs one of India's most active student internship programs — it should be a top-level nav item, not buried.

UX Audit — Existing NayePankh Site

Analysis of current issues and recommended improvements.

Issues Found

High priority
No beneficiary pathway
The site assumes every visitor is a potential volunteer or donor. Students and parents who need help have no clear path. A "Get Help" CTA is missing entirely.
High priority
No dedicated internships page
NayePankh runs one of India's most active student internship programs — yet there is no top-level page for it. Applicants have to search externally to find the tasks.
Medium priority
Hero section lacks emotional pull
The current hero uses generic NGO stock imagery and a minimal headline. It doesn't immediately communicate who is being helped, or why that should matter to the visitor.
Medium priority
No impact numbers above the fold
Stats like "50,000+ students reached" build trust fast. They are currently buried mid-page or absent. Placing them in the hero dramatically improves credibility for new visitors.
Low priority
Mobile nav is hard to use
On mobile, the navigation collapses but the tap targets are small. Primary CTAs like "Volunteer" should persist as a sticky bottom bar on mobile for easier access.
Low priority
Colour contrast issues
Some text on coloured backgrounds does not meet WCAG AA contrast ratio (4.5:1 minimum). Particularly grey text on light backgrounds. Needs audit and fixes.
